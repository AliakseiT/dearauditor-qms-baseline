## 1. Purpose
Define repeatable execution for V&V planning, test execution, defect handling, and release evidence packaging.

## 2. Scope
Applies to all product changes with potential quality, safety, or regulatory impact.

## 3. Procedure
1. Define V&V scope from requirements, risk controls, and change impact.
2. Create or update test protocol with acceptance criteria and objective evidence format.
3. Execute verification activities (reviews, static checks, automated/manual tests).
4. Execute validation activities for intended use and user needs where applicable.
5. Record failed results as defects/nonconformities and link them to CAPA where systemic.
6. Re-test after fixes and document pass/fail disposition.
7. Prepare V&V summary with trace links (requirement -> test -> result -> disposition).
8. Block release until all critical V&V acceptance criteria are met or formally justified.

## 4. Required Records
- V&V plan/protocol
- Raw execution results and evidence attachments
- V&V summary report with release recommendation
